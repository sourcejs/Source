Linking Your Project's CSS

Your project's CSS should be linked to each spec page to render your examples, and you can also add styles specific to a spec page:

<link href="/data/docs/project.css" rel="stylesheet"> <link href="css/spec.css" rel="stylesheet">

Spec Page Markup

Documentation Text

Only the following tags in documentation sections are styled by Source:

  • p
  • h1
  • h2
  • h3
  • h4
  • ul
  • li

This styling is not applied inside example blocks.

Markdown support for editing documentation sections will be available within the next few updates.

HTML Head

After starting your Node.js server, common markup, such as the header, footer, and inner navigation, are dynamically included by the server.

Metas

Machine readable information - used by plugins such as Global Navigation and Search - can be added via the following meta tags:

<meta name="keywords" content="create spec, spec markup"> <meta name="author" content="Robert Haritonov">

Meta information can also be used to separate page roles:

<meta name="source-page-role" content="navigation">

The example above will indicate to Source that the page's role is for navigation, and not for use as a spec. Inner navigation, which generates a table of contents on spec pages, will not be output on pages with this meta tag. You will find this meta tag used on the Source.js home page.
